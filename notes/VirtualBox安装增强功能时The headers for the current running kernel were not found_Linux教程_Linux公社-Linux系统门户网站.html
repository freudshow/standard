VirtualBox安装增强功能时The headers for the current running kernel were not found

[日期：2014-05-22] 来源：Linux社区  作者：Linux [字体：大 中 小]

环境：主系统：windows XP
 
          virtualbox版本：4.3.8
 
          guest系统：Ubuntu12.04.4
 
现在官网上只能下到ubuntu12.04.4，听说virtualbox的版本对ubuntu增强的安装影响很大，网上查资料说对ubuntu12.04至少得用vbox4.2.12，结果下载安装后，在安装增强时出错：
 
Removing existing VirtualBox DKMS kernel modules ...done.
 Removing existing VirtualBox non-DKMS kernel modules ...done.
 Building the VirtualBox Guest Additions kernel modules
 The headers for the current running kernel were not found. If the following
 module compilation fails then this could be the reason.
 
Building the main Guest Additions module ...done.
 Building the shared folder support module ...failed.
 

......
 
网上搜原因，有说 缺少kernal的头文件
 
于是按链接的方法处理
 
sudo apt-get install dkms build-essential linux-headers-$(uname -r)
sudo /etc/init.d/vboxadd setup

结果还是没反应，继续搜，有人说virtualbox版本过低有影响，于是提高版本为virtualbox4.2.20，结果还是一样。。尼玛，头大了
 
于是一狠心继续提版本 virtualbox4.3.8，心想这次再不行就是ubuntu的内核版本问题了吧，已经做好了重下ubuntu的准备。。
 
安装好virtualbox4.3.8后，再按上文方法处理，结果成功了！
 
这些东西的各个版本很奇怪，我明明网上看有人用4.2.12也能安装成功，不解不解。。不过多尝试总是能成功的= =
